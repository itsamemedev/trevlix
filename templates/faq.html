Haeufig gestellte Fragen

Antworten auf die wichtigsten Fragen rund um TREVLIX

18 Fragen · Klicke auf eine Frage zum Aufklappen

Allgemein

Was ist TREVLIX? ▼

TREVLIX ist ein Open-Source Algorithmic Trading Bot fuer Kryptowaehrungen. Er nutzt ein 4-Modell KI-Ensemble mit 9 Voting-Strategien, um automatisch Kauf- und Verkaufsentscheidungen zu treffen.

Ist TREVLIX kostenlos? ▼

Ja, TREVLIX ist vollstaendig Open Source unter der MIT-Lizenz. Du kannst es kostenlos nutzen, modifizieren und weiterverbreiten. Es gibt keine versteckten Kosten oder Premium-Versionen.

Welche Exchanges werden unterstuetzt? ▼

Aktuell werden 5 Exchanges unterstuetzt: Crypto.com, Binance, Bybit, OKX und KuCoin. Die Integration erfolgt ueber die CCXT-Bibliothek, sodass weitere Exchanges leicht hinzugefuegt werden koennen.

Kann ich TREVLIX auf einem Raspberry Pi betreiben? ▼

Ja! TREVLIX laeuft auf Raspberry Pi 4 (4GB+ RAM empfohlen). Die LSTM/TensorFlow-Komponente ist optional und kann bei begrenztem RAM deaktiviert werden. Das 3-Modell-Ensemble (ohne LSTM) laeuft problemlos auf dem Pi.

Trading & Strategie

Wie funktioniert das Voting-System? ▼

9 technische Strategien (EMA, RSI, MACD, Bollinger, Volume, OBV, ROC, Ichimoku, VWAP) stimmen gleichzeitig ab. Jede gibt BUY, SELL oder HOLD. Ueberwiegt ein Signal den konfigurierbaren Schwellenwert (Standard: 5/9), wird ein Trade ausgefuehrt. Die KI gewichtet die Stimmen dynamisch basierend auf historischer Performance.

Was ist Paper Trading? ▼

Paper Trading ist ein Simulationsmodus, der echten Handel nachahmt, aber kein echtes Geld einsetzt. Es ist dringend empfohlen, mindestens 30 Tage im Paper Trading Modus zu testen, bevor man mit echtem Geld handelt. Setze PAPER_TRADING=true in der .env-Datei.

Kann TREVLIX auch Short-Selling? ▼

Ja, auf Exchanges die Futures/Margin unterstuetzen (Binance Futures, Bybit). Der Bear-Regime KI-Modell ist speziell fuer Short-Positionen optimiert. Short-Selling wird nur aktiviert, wenn der Markt in einem Abwaertstrend ist und die KI eine hohe Konfidenz hat.

Wie hoch ist die erwartete Rendite? ▼

Renditen haengen stark von Marktbedingungen, Konfiguration und Exchange ab. TREVLIX gibt keine Renditeversprechen. In Backtests zeigen die Strategien typischerweise Win-Rates von 55-65%. Vergangene Performance garantiert keine zukuenftigen Ergebnisse. Starte immer mit Paper Trading!

Was ist der Circuit Breaker? ▼

Der Circuit Breaker ist ein Sicherheitsmechanismus, der den Handel automatisch pausiert, wenn zu viele aufeinanderfolgende Verluste auftreten oder das taegliche Verlustlimit erreicht wird. Er schuetzt vor unkontrollierten Verlusten in volatilen Maerkten und kann in den Einstellungen konfiguriert werden.

KI & Machine Learning

Wie lernt die KI? ▼

Die KI lernt aus vergangenen Trades. Nach 30+ Trades beginnt das Training, danach wird nach jedem 5. Trade neu trainiert. Das System nutzt Walk-Forward Validation, um Overfitting zu vermeiden. Vier Modelle (Global, Bull-Regime, Bear-Regime, LSTM) bilden ein Ensemble fuer robustere Vorhersagen.

Brauche ich eine GPU fuer die KI? ▼

Nein, die Modelle (Random Forest, XGBoost, LightGBM, CatBoost) laufen effizient auf CPU. Das optionale LSTM-Modell profitiert von einer GPU, ist aber auch ohne GPU nutzbar. Fuer die meisten Nutzer reicht ein Standard-Server oder Raspberry Pi 4.

Was sind die 31 Features des ML-Modells? ▼

Die Features umfassen: 7 Strategie-Votes, RSI normalisiert, Stochastic RSI, Bollinger %B und Width, MACD-Histogramm (Vorzeichen + Steigung), Volume Ratio, ATR%, EMA-Alignment, Preis vs. EMA21, ROC-10, Bull/Bear-Flag, Stunde (Sinus/Cosinus-codiert), Vote-Konsensus und die juengste Win-Rate der letzten 10 Trades.

Installation & Betrieb

Welche Systemvoraussetzungen hat TREVLIX? ▼

Python 3.9+ (empfohlen 3.11), MySQL 8.0+, 2 GB RAM (4 GB mit LSTM), Ubuntu 20.04+/Debian 11+/macOS/Windows (WSL). Docker wird fuer die einfachste Installation empfohlen.

Wie update ich TREVLIX? ▼

TREVLIX hat ein eingebautes GitHub Auto-Update System. Im Dashboard kann ein Update mit einem Klick gestartet werden. Alternativ: cd trevlix && git pull origin main. Bei Docker: docker compose pull && docker compose up -d.

Wie sichere ich meine API-Keys? ▼

API-Keys werden automatisch mit Fernet-Verschluesselung in der Datenbank gespeichert. Setze ENCRYPTION_KEY in der .env-Datei (wird beim Setup generiert). Nutze IP-Whitelisting bei deiner Exchange und beschraenke die API-Berechtigungen auf das Minimum (nur Trading, kein Withdrawal).

Sicherheit & Risiko

Ist TREVLIX sicher? ▼

TREVLIX implementiert umfangreiche Sicherheitsmassnahmen: JWT-Authentifizierung, 2FA/TOTP, bcrypt-Passwort-Hashing, Fernet-API-Key-Verschluesselung, Rate-Limiting, CORS-Schutz und IP-Whitelisting. Alle Aktionen werden im Audit-Log protokolliert. Fuer maximale Sicherheit nutze HTTPS via Nginx.

Kann ich Geld verlieren? ▼

Ja. Krypto-Trading birgt immer Verlustrisiken. TREVLIX minimiert Risiken durch Circuit Breaker, Stop-Loss, und taegliche Verlustlimits — aber kein System kann Verluste vollstaendig verhindern. Investiere nur, was du dir leisten kannst zu verlieren. Starte immer mit Paper Trading!

Wo bekomme ich Hilfe? ▼

Bei Problemen oder Fragen: (1) Pruefe die Installationsanleitung, (2) Erstelle ein GitHub Issue, (3) Tritt dem Discord-Server bei. Fuer Sicherheitsprobleme nutze die verantwortungsvolle Offenlegung ueber GitHub Security Advisories.
