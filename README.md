## Features

- **Multi-Exchange Support** ‚ÄĒ Crypto.com, Binance, Bybit, OKX, KuCoin simultaneously
- **14+ AI Modules** ‚ÄĒ XGBoost, LightGBM, CatBoost, LSTM, Transformer, Random Forest
- **9 Voting Strategies** ‚ÄĒ EMA Trend, RSI+Stochastic, MACD, Bollinger, Volume, OBV, ROC, Ichimoku, VWAP
- **Real-time Dashboard** ‚ÄĒ WebSocket-powered live UI with Socket.io
- **Grid Trading** ‚ÄĒ Automated grid strategies with configurable levels
- **Monte-Carlo Risk Analysis** ‚ÄĒ Portfolio simulations with VaR calculation
- **Circuit Breaker** ‚ÄĒ Automatic trading pause after consecutive losses
- **Telegram Notifications** ‚ÄĒ Real-time alerts for all trades
- **Discord Integration** ‚ÄĒ Webhooks with daily reports
- **Audit Log** ‚ÄĒ Full action history with timestamps
- **Break-Even Stop-Loss** ‚ÄĒ Automatic SL adjustment after profit
- **Symbol Cooldown** ‚ÄĒ Locks symbols after a loss
- **IP Whitelist** ‚ÄĒ Access control by IP address
- **News Sentiment Filter** ‚ÄĒ Blocks trades on negative news
- **Funding Rate Filter** ‚ÄĒ Avoids expensive short positions
- **Paper Trading** ‚ÄĒ Risk-free testing without real capital
- **Copy Trading** ‚ÄĒ Followers receive all signals in real time
- **Multi-User System** ‚ÄĒ Multiple portfolios on a single instance, each with their own API keys
- **2FA** ‚ÄĒ Two-factor authentication (TOTP)
- **5 Languages** ‚ÄĒ DE, EN, ES, RU, PT
- **Full REST API** ‚ÄĒ JWT-authenticated API with WebSocket real-time updates
- **GitHub Updater** ‚ÄĒ In-dashboard one-click update & rollback

## Quick Installation

```bash
# One-liner install (Ubuntu/Debian)
curl -O https://raw.githubusercontent.com/itsamemedev/Trevlix/main/install.sh
sudo bash install.sh
```

Or manually:

```bash
git clone https://github.com/itsamemedev/Trevlix.git
cd Trevlix
pip install -r requirements.txt
cp .env.example .env
nano .env          # Enter API keys
python server.py
```

Dashboard: **http://localhost:5000**

## Configuration

All settings in `.env`:

```env
# Exchange
EXCHANGE=cryptocom
API_KEY=your_api_key
API_SECRET=your_secret

# Security
ADMIN_PASSWORD=secure_password_min_12_chars
JWT_SECRET=random_hex_string_32chars
SECRET_KEY=random_hex_string_32chars
ENCRYPTION_KEY=fernet_key  # python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# CORS (optional ‚ÄĒ default allows all origins)
# For production, restrict to your domain:
# ALLOWED_ORIGINS=https://yourdomain.com,http://localhost:5000

# Trading
PAPER_TRADING=true   # Always start in paper mode first!

# Notifications (optional)
DISCORD_WEBHOOK=https://discord.com/api/webhooks/...
TELEGRAM_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id
```

## WebSocket / Dashboard API

The dashboard communicates with the server exclusively via Socket.io. All commands are sent as events:

| Event | Direction | Description |
|-------|-----------|-------------|
| `start_bot` / `stop_bot` / `pause_bot` | Client ‚Üí Server | Bot control |
| `update_config` | Client ‚Üí Server | Save settings |
| `save_api_keys` | Client ‚Üí Server | Store encrypted exchange keys |
| `close_position` | Client ‚Üí Server | Manually close a trade |
| `run_backtest` | Client ‚Üí Server | Start backtest |
| `force_train` / `force_optimize` | Client ‚Üí Server | Trigger AI training |
| `check_update` / `apply_update` / `rollback_update` | Client ‚Üí Server | GitHub updater |
| `start_exchange` / `stop_exchange` | Client ‚Üí Server | Multi-exchange control |
| `save_exchange_keys` | Client ‚Üí Server | Save per-exchange API keys |
| `scan_arbitrage` | Client ‚Üí Server | Find price spreads |
| `create_grid` | Client ‚Üí Server | Create grid strategy |
| `update` | Server ‚Üí Client | Full state snapshot |
| `status` | Server ‚Üí Client | Toast notification |
| `trade` | Server ‚Üí Client | Trade executed |
| `ai_update` | Server ‚Üí Client | AI model update |
| `backtest_result` | Server ‚Üí Client | Backtest completed |
| `price_alert` | Server ‚Üí Client | Price alert triggered |
| `update_status` | Server ‚Üí Client | GitHub update info |

## Multi-User & API Keys

Each registered user stores their own exchange API keys in the database (Fernet-encrypted). The bot runs trades for each user using their personal credentials ‚ÄĒ no shared keys, full separation. An admin can manage users and global bot configuration via the admin panel.

## Security Notes

- **CORS**: By default `ALLOWED_ORIGINS=*` (all origins allowed) for easy local setup. **Set a specific origin in production** via `.env`.
- **Session auth**: Dashboard requires login. WebSocket connections are rejected if not authenticated.
- **Admin-only**: Sensitive actions (apply update, manage users, exchange keys) require `role=admin`.
- **Encryption**: All API keys are Fernet-encrypted before storage.
